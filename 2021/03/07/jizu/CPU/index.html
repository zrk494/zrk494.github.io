CPU

CPU功能和基本结构

结构
  • 运算器
    • ALU（算数逻辑单元）
    • ACC（累加寄存器）
    • PSW（程序状态字寄存器）：在使用条件转移指令会用到
    • 移位器+计数器
    • 暂存寄存器（Y）
    • 通用寄存器
  • 控制器（控制单元CU）
    • PC（程序计数器）
    • IR（指令寄存器）
    • MAR（存储器地址寄存器）
    • MDR（存储器数据寄存器）
    • 指令译码器+时序系统+微操作信号发生器
  • 数据通路：专用通路+内部总线
  • 补充：
    • 汇编程序员可见：通用寄存器+PSW+PC+ACC
    • 不可见：IR+DR（暂存寄存器）+MAR+MDR
功能
  • 运算器：加工数据
  • 控制器：取指令，分析指令，执行指令
  • 总体来看CPU的功能：指令控制+操作控制+时间控制+数据加工+中断处理

CPU中指令的执行过程

指令周期相关基本概念
  • 重点区分：指令周期/机器周期/时钟周期

  • 指令周期：即CPU从主存中取出一条指令的全部时间

    • 包括若干个机器周期（CPU周期）
    • 机器周期包括多个时钟周期（节拍，T周期）

  • 时钟周期是CPU操作的基本单位

  • 每个指令周期内的机器周期可不等，每个机器周期内的节拍数也可以不等

指令周期之中的数据流向

  • 取址周期（完成取指令，标志位为FE）：

    1. (PC)—>MAR：将当前地址送至存储器地址寄存器
    2. 1—>R：CU发出控制信号，经过控制总线传到主存
    3. M(MAR)—>MDR：取出MAR中的地址所指内容，通过数据总线传送给MDR
    4. (MDR)—>IR：将MDR中的内容（一条指令）送入IR
    5. (PC)+“1”—>PC：CU发出控制信号，形成下一条指令地址

  • 间址周期（完成有效地址转换，标志位为IND）

  • 执行周期（完成取操作数，标志位为EX）：具体问题具体分析，无统一的情况
  • 中断周期（保存程序中断点，标志位为INT）

注：通过这个过程中各个寄存器，部件以及三条线完整描述数据的流向

数据通路

概述
  • 数据在功能部件之间传送的路径
  • 主要研究信息从哪里开始+中间经过哪里+最后传到哪里
  • 在一个指令的过程中关注哪些控制信号（控制数据输入+控制数据输出）是有效的
单总线结构

  • 只通过CPU内部总线进行数据交换，两个部件独占

  • 寄存器之间的数据传送

    1. (PC)—>Bus（Bus表示总线）
    2. Bus—>MAR

  • 主存与CPU之间的数据传送

  • 执行算数或者逻辑运算

    • 注：这种结构下必须先将其中一个操作数先放到暂存寄存器中

  • 注：对于指令ADD (R0),R1:

    • 不带括号表示数直接放在R1对应的寄存器中，带括号要进行一次寄存器间接寻址
    • R0是目的操作数，对于加法指令，它会存放最终结果地址（即指向结果所在的内存位置）

控制器

硬布线控制器
  • 微操作控制信号通过组合逻辑电路根据当前的状态（PSW），指令码和时序产生
  • 特点
    • 逻辑线路固定，难以扩充和修改
    • 电路结构复杂
    • 通过电路硬件实现功能，速度快
微程序控制器
  • 采用存储程序的原理，将微操作控制信号以程序的形式存放在存储器之中
  • 一条指令对应一个微程序，执行指令时将其从内存中读出即可
  • 特点：
    • 本质上是对指令的封装
  • 微指令编码方式

指令流水线

划分
  • 一般情况下，一条指令的执行过程按照功能划分成4个阶段：取指令+译码+执行+存结果
  • 时钟周期主要依据耗时最长的过程
指令流水线的阻塞因素
  1. 因为结构相关的原因发生冲突：比如两条指令在同一时刻访问内存
    • 解决方法：后续的指令等一个周期再去访存/单独设置指令存储器和数据存储器
  2. 后续的指令需要的数据依赖之前的指令
    • 解决方法：后面的指令等一段时间/数据旁路技术（转发机制），即把前一条指令在ALU的输出结果作为之后指令在ALU之中的输入数据
  3. 与控制相关，即流水线遇上转移指令之类的让PC发生改变的指令，造成断流
