计算机存储系统

发表于 2021-03-07 更新于 2022-12-01 分类于 计算机组成原理

    • 计算机存储系统

      概述

      层次结构
      1. 第一层：CPU寄存器
      2. 第二层：Cache（高速缓存）
      3. 第三层：主存储器
      4. 第四层：辅存（外存，如磁盘）
      分类（多种分类的方式）
      • 按层次结构
      • 按存储介质
      • 按存取方式
      • 按信息是否可以更改
      • 断电后信息是否消失
      • 信息读出后，原信息是否被破坏（如DRAM就是一种破坏性读出的芯片）
      存储器的性能指标
      • 存储容量=存储字数*字长
      • 单位成本
      • 数据传输率（即主存带宽）=数据的宽度/存储周期
      • 存储周期（一般有两个部分）=存取时间+恢复时间

      主存储器

      主存的基本构成

      • 基本元件

        • MOS管

        • 电容，存储电荷（以这种方式存0/1）

      • 存储芯片的结构

        • 译码驱动电路

        • 存储矩阵（即存储体部分）

        • 读写电路

        • 地址线：对应地址的位数

        • 数据线：对应数据的位数

        • 片选线

        • 读写控制线

      • 寻址

        • 通常按字节编址，即每个字节对应一个地址

        • 还可以按字寻址，按半字寻址，按双字寻址

      SRAM和DRAM

      • RAM芯片都是易失性的存储器，断电后信息丢失

        • SRAM是非破坏性读出，不需要刷新，常用于Cache和TLB，集成度较低（双稳态触发器）
        • DRAM是破坏性读出，要定期刷新，常用于主存，集成度较高（电容）

      • DRAM的刷新

        • 周期一般为2ms，以行为单位

      • DRAM的地址线复用技术

        • DRAM的地址包括行地址和列地址（参照二维数组）
        • 通过复用技术不再有行地址译码器和列地址译码器，一个译码器即可，分两次选，以节约地址线
##### ROM（非重点）

- 非易失性存储，可用于主存，来存BIOS

- MROM
- PROM
- EPROM
- Flash Memory，支持随机存取
- SSD（Solid State Drives）

##### 主存的优化

- 低位交叉存储器
  - 每个存储体组合起来时，每个体读或写完之后需要一定的恢复时间
  - 地址包括两部分：高位是体内地址，低位是体号，这样两个相邻的地址号就对应不同的存储体
  - ![image-20221116210924654](存储系统/image-20221116210924654-16698592444851.png)
  - 一些特点
    - 每个模块有相同的容量和存取速度
    - 各个模块有独立的读写控制电路，地址寄存器以及数据寄存器（对于一个地址，先分析其体号，在去对应体的地址寄存器找对应的内容）
    - 可以采用流水线的模式访问存储体提高效率
    - 如果模块存取一个字的周期为T，总线传送周期为r，为了保证流水线不间断，要保证存储体交叉模块数（存储体数量）$m\ge T/r$
    - 地址对总块数取余可得到所在的块号


#### 主存储器与CPU的连接

- 位扩展：数据总线宽带>存储芯片字长
- 字扩展：扩展主存字数
- 数据总线传数据，几根线对应几位数（地址总线类似）
- 内存容量的表述方式，如8k$\times$1位：表示8k个存储单元，每个单元1位数，地址线13根，数据线1根，二者相乘即为芯片容量

##### 单块存储芯片连接CPU

- CPU的地址线和数据线与内存芯片相同即可

##### 多块存储芯片连接CPU

- 位扩展法：增加主存的存储字长，是其数据位数与CPU的数据线数相同（扩展存储单元的容量）
  - 这种情况下，主存的数据线<CPU的数据线（数据位数）
  - 例：![image-20221028163104674](存储系统/image-20221028163104674-16698592444862.png)


- 字扩展法：增加主存中字的数量（存储单元的数量），位数不变
  - 这种情况下CPU和主存芯片的数据线相同，CPU地址线>主存地址线.
  - 线选法：用多余的地址线**直接**连接多块相同的存储芯片，**高位**就可以表示是哪一片芯片（直接连接的灵活性差，且存在被浪费的位）
  - 译码片选法：用多余的的地址线**连接译码器**，再连接对应地址芯片（保证高位充分利用）
- 字位同时扩展（综合前两种）：具体问题具体分析

- 芯片数=容量/单片芯片容量

#### Cache

##### 概述

- 工作原理：将主存中的部分内容复制到Cache中，缓和CPU和主存之间的矛盾
- 局部性原理
  - 空间局部性：现在访问的地址，其附近的地址也很可能积极被访问
  - 时间局部性：现在访问的地址，不久之后很可能再次被访问
- 主要性能指标：命中率
  - 注：根据命中率计算分析性能（时间）
  - 计算平均访问时间
- Cache的两种方式
  - 先访问Cache，再访问主存
  - 同时访问主存和Cache
  - 注意二者平均访问时间的计算
- 主存和Cache以块为单位进行数据交换（主存和Cache使用同一种分块方式，即他们的分块大小都相等）
  - 分块之后，将主存地址划分成两部分：块号+块内地址
  - 注：块号从0开始

##### Cache和主存的映射方式

- Cache中存的内容
  - 有效位：0无效，1有效，用于初始化
  - 脏位
  - 替换控制位
  - 标记：此块数据在内存中的实际地址（两部分：主存块号+快内地址）
    - 注：在组相联映射下的还有组号部分
  - 整块数据
- 全相联映射
  - 主存块在Cache中随便放
- **直接映射**
  - 主存块放到特定的某个Cache行，行号=主存块号%Cache总行数
  - 标记优化（提高查找的效率）
  - 地址结构：主存字块标记（区分是哪一个存储块，）+Cache字块地址（即数据存在哪一个Cache块中，根据Cache的总块数决定）+字块内地址
- **组相联映射**
  - 将Cache中的块分组，通过$主存块号\%组数$得到主存中某一块放到哪一组
  - 地址结构：主存字块标记+组地址（数据在哪一组）+字块内地址
  - 注：2路组相联，表示每组两块

##### Cache替换策略

- 就是Cache满了之后，要将哪些数据替换出去
- 随机法（RAND）
- 先进先出法（FIFO）
- **近期最少使用法**（LRU）
  - 每个Cache块会设置一个“计算器”，用于记录Cache多久没被访问过，
  - 注意标志位，有效位和LRU算法标志位
- 最不常使用法（LFU）

##### Cache的写策略

- 全写法（write-through）
- 写回法（write-back）
  - 这个方法要在Cache块中添加一个脏位，用来表示此Cache块是否被修改过



- 注：综合Cache相关的3个方面来计算Cache的每一块应该占多少位
  - 不要忘记有效位，脏位等情况
