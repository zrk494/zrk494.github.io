操作系统---概述

发表于 2021-03-07 更新于 2021-06-09

操作系统—导论

功能

1. 用户视角

使用方便，不在乎资源利用

2. 系统视角

资源分配（cpu时间，内存空间，文件存储空间，i/O设备等）

定义（三方面）

  1. 资源管理平台
  2. 运行程序的平台
  3. 为用户提供服务的平台

计算机系统

1. 启动

1. BIOS(basic input output system)
2. bootstrap program(引导程序)
  • 在通电后运行
  • 不是操作系统的一部分
  • 作用包括—1.初始化设备 2.定位到操作系统，并将其载入内存 3.运行系统的首个进程init（），并等待事件发生

    2. 中断

1. 中断用于下达指令通知事件的发生（比如鼠标点击打开一个应用就是一个中断)
2. 硬中断和软中断
3. 发生中断的时候CPU会运行中断服务程序
4. 中断向量（保存着中断程序的地址）
5. 过程
  • 发生中断时，系统保存当前进程的程序计数器（PC）
  • CPU跳转到中断服务程序程序中，进行中断服务
  • 中断服务结束后，CPU返回到被中断的程序（通过PC找到中断的位置），程序继续向下进行

3. I/O

  • 软中断，硬中断

  • 设备控制器

    • 存在自己的本地缓冲器和寄存器
    • 负责相应类型的设备控制

  • I/O和CPU可同时运行

  • I/O有自己的本地缓冲器和寄存器

  • DMA

    • 在没有CPU的干涉下，通过专门的硬件（DMA控制器），以块为单位完成设备缓冲器与主存之间的数据批量传送
    • 减少了CPU对I/O的干预

  • 完成I/O的过程

    • CPU在内存和本地缓冲器之间交换数据
    • 设备控制器控制外设和本地缓冲之间的数据交换
    • I/O操作结束之后，设备控制器通过中断通知CPU已经完成I/O

  • I/O的同步操作

  • I/O的异步操作

4. 计算机系统的体系结构

4.1单处理器系统
  • 系统中只有一个通用处理器（CPU）
  • 还存在一些专用处理器
4.2多处理器系统
  • 系统有多个处理器，这些CPU共享一个内存
  • 优点
    • 增加了吞吐量
    • 方便扩展
    • 增加了可靠性
  • 非对称处理器
  • 对称处理器
  • 多处理器系统中，每个CPU处理不同的数据，要将这些数据（如PC等）存放在自己的寄存器中，同时也需要有自己的高速缓存与寄存器中的内容对应
4.3集群系统
  • 一组相互连接的计算机组成的分布式系统
  • 各个自治的系统协同工作（拥有自己独立的操作系统）

操作系统的结构

1. 结构

1.1 批处理系统

CPU按顺序执行程序，请求调用I/O之后，会等待I/O工作完成

1.2 多道程序系统（因为I/O的运行时间长，所以以此方式提高CPU利用率）

有多个程序同时在内存中，CPU等待I/O时，会去执行其他的程序

1.3 分时系统（让每个任务可以公平的使用CPU）

每个任务被赋予一个时间片，CPU在多个任务之间切换（以时间片为周期）

2. 操作相关

  • 要区分出操作系统的代码和用户代码，所以操作系统提供了双重模式操作
  • 用户模式（运行用户代码）和内核模式（运行系统代码），二者通过模式位区分

3. 管理内容

  • 进程管理
  • 内存管理
  • 文件系统管理
  • I/O相关

5. 系统调用

5.1定义相关

操作系统服务的编程接口（用户和计算机硬件之间），用c或者c++或者汇编语言编写。

5.2 为什么需要系统调用

在用户程序之中，凡是与系统资源相关的操作，都要通过系统调用的方式向操作系统发出请求，由操作系统代完成，以 保障系统稳定安全

5.3按功能分类

设备控制，文件控制，进程控制，进程通信，内存管理

5.4API，系统接口

程序通常通过API访问，不直接使用系统调用
