关系数据库

1.关系

1.1 相关概念

1.1.1 属性
  • 关系表的列
  • 列是同质的，即为同一数据类型
1.1.2 域

相同类型数据的集合

1.1.3 关系

笛卡尔集的子集（详见集合论与图论）

1.1.4 关系模式

关系的形
记作 R（U）,U为A1…An组成的属性集合，R是关系名
例：Student（学号，姓名，性别，年龄，专业）
Grade（学生编号，课程编号，成绩）

1.1.5 关系实例

一个关系的任何一个“当前值”是该关系的实例

1.1.6 元组

笛卡尔积得到的集合中的每一个元素

1.1.7 码（又称作键， 那个名为篝酱的少女）
超码

可以==唯一==标识一个元组的一个属性集合

候选码

唯一的一个属性集合或者属性集合组，超码是候选码的特殊表现，可存在多个

主码

数据库设计者选择的用于区分不同元组的候选码（从候选码之中选出一个）

外码

非关系R的候选码，而是关系S的候选码，将两个关系联系在一起

1.1.8 关系的第一范式

属性不可再分

1.2 关系的完整性

1.2.1 实体完整性约束

主键属性不空（不存在或者无意义）

1.2.2 参照完整性约束（连接两个关系）

外码在R1之中的值为空或者对应其在R2（在R2之中为主码）之中的值

1.2.3 用户定义的完整性

  • 笛卡尔积 表与表之间 $\times$

  • 选择 同一个表 $\sigma$ (简单理解为select)
  • 投影 同一个表 $\prod$
  • 重命名 同一个表 $\rho$
  • 除法
  • 连接
  • 等值连接
  • 自然连接
  • 外连接
  • 全外连接

2. SQL语言基础

结构化查询语言，关系数据库的标准语言

操作的对象是集合，之后得到的结果也是集合

2.1 基本的9个引导词

  • 数据定义 create ,drop,alter
    查询 select
    修改 insert,update,delete
    数据控制 grant,revote

示例数据库(带下划线的为主键)

Student(Sno,Sname,Ssex,Sage,Sdept)

Course(Cno,Cname,Cpno,Ccredit)

SC(Sno,Cno,Grade)

2.2 数据定义

2.2.1 create（创建，包括表，视图—一种虚拟的表，索引）

  • create table

    语法格式 ：

    create table <关系名> (<属性名> <属性类型>[NOT NULL], … , <属性名> <属性类型>[NOT NULL], 完整性约束1，完整性约束2,…)

    实例：

  • create index(为加快查询而定义的)

    语法格式：CREATE [UNIQUE] INDEX 〈索引名〉 ON 〈关系名〉(〈列名 〉[ORDER], …,〈列名〉[ORDER]) [CLUSTER]

    其中 [unique]，表示索引属性是不能重复出现的，还可以是[cluster]

    order有两种值 ASC(升序)或者DESC(降序)

    create unique index NUM on Game(NUM ASC) cluster

2.2.2 drop (删除)

  • drop table（删除整个表）

    语法格式：

    drop table <关系名>

  • drop index

    语法格式：

    drop index<索引名>

  • drop

2.2.3 alter （修改）

  • alter table（增加或减少一列）

    语法格式：

    alter table <关系名> ADD(增加)|DROP(删除)<列名><列类型>

  • alter index <旧索引名>rename to <新索引名>

3.数据库安全性，完整性（用SQL实现）

3.1 基本概念

3.1.1安全性
  • 不允许恶意修改或者未经授权的存取和修改
3.1.2完整性（为保障数据正确性，相容性，添加的约束条件和规则）
  • 语义完整性（重点）
  • 并发控制，安全控制，DB回复等

3.2 控制安全性

3.2.1授权

语法格式

GRANT <权限> [, 权限] …

ON <对象类型> <对象名> [, <对象类型> <对象名> ] …

TO <用户> [,用户] … [WITH GRANT OPTION];

例

Grant select

on table game

to krzkrz;

3.2.2 权限回收

REVOKE <权限> [, 权限] …

ON <对象类型> <对象名> [, <对象类型> <对象名>]

FROM <用户> [, 用户] … [CASCADE | RESTRICT]

常用方法

    • 主体和客体
    • 敏感度
    • 只能修改（写）同级或者更高级（此时数据未泄漏）的相关数据
