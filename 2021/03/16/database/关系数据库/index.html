关系数据库

1.关系

1.1 相关概念

1.1.1 属性
  • 关系表的列
  • 列是同质的，即为同一数据类型
1.1.2 域

相同类型数据的集合

1.1.3 关系

笛卡尔集的子集（详见集合论与图论）

1.1.4 关系模式

关系的形
记作 R（U）,U为A1…An组成的属性集合，R是关系名
例：Student（学号，姓名，性别，年龄，专业）
Grade（学生编号，课程编号，成绩）

1.1.5 关系实例

一个关系的任何一个“当前值”是该关系的实例

1.1.6 元组

笛卡尔积得到的集合中的每一个元素

1.1.7 码（又称作键， 那个名为篝酱的少女）
超码

可以==唯一==标识一个元组的一个属性集合

候选码

唯一的一个属性集合或者属性集合组，超码是候选码的特殊表现，可存在多个

主码

数据库设计者选择的用于区分不同元组的候选码（从候选码之中选出一个）

外码

非关系R的候选码，而是关系S的候选码，将两个关系联系在一起

1.1.8 关系的第一范式

属性不可再分

1.2 关系的完整性

1.2.1 实体完整性约束

主键属性不空（不存在或者无意义）

1.2.2 参照完整性约束（连接两个关系）

外码在R1之中的值为空或者对应其在R2（在R2之中为主码）之中的值

1.2.3 用户定义的完整性

1.3 关系运算相关（重点）

2. SQL语言基础

2.1 基本的9个引导词

数据定义 create ,drop,alter
查询 select
修改 insert,update,delete
数据控制 grant,revote

2.2 数据定义

2.2.1 create（创建，包括表，视图，索引）

  • create table

    语法格式 ：

    create table <关系名> (<属性名> <属性类型>[NOT NULL], … , <属性名> <属性类型>[NOT NULL], 完整性约束1，完整性约束2,…)

    实例：

    create table Game

    ( Name varchar(10) not null,//长度可变的字符串,且非空

    NUM char(10) not null,

    Type char(5),//定长字符串

    year integer,

    Primary key (NUM))

  • create index

    语法格式：CREATE [UNIQUE] INDEX 〈索引名〉 ON 〈关系名〉(〈列名 〉[ORDER], …,〈列名〉[ORDER]) [CLUSTER]

    其中 [unique]是固定的，表示索引属性是不能重复出现的,order有两种值 ASC或者DESC

    create unique index NUM on Game(NUM ASC) cluster

2.2.2 drop (删除)

  • drop table（删除整个表）

    语法格式：

    drop table <关系名>

  • drop index

    语法格式：

    drop index<索引名>

2.2.3 alter （修改）

  • alter table（增加或减少一列）

    语法格式：

    alter table <关系名> ADD|DROP<列名><列类型>

2.3 查询

2.3.1 单表查询（只有一个表，一般为查找一个表的一列或者多列）

​ 有一个表 Game(Gname, Gsize, Gtype, Gyear)

  • 查询指定列(例子中的名称和大小)

    Select Sname, Ssize

    From Game;

  • 查询全部

    Select *

    From Game;

  • 查符合条件的元组

    where子句( between and, not between and, in , not in, like, not like, and, or, is null)

    例1 2001到2020年发行的游戏的名称

    Select Sname

    From Game

    Where year between 2001 and 2020;

  • 聚集函数（6种）

    例4 统计游戏个数

3.数据库安全性，完整性（用SQL实现）

3.1 基本概念

3.1.1安全性

  • 不允许恶意修改或者未经授权的存取和修改

3.1.2完整性（为保障数据正确性，相容性，添加的约束条件和规则）

  • 语义完整性（重点）
  • 并发控制，安全控制，DB回复等

3.2 控制安全性

3.2.1授权

语法格式

GRANT <权限> [, 权限] …

ON <对象类型> <对象名> [, <对象类型> <对象名> ] …

TO <用户> [,用户] … [WITH GRANT OPTION];

例

Grant select

on table game

to krzkrz;

3.2.2 权限回收

REVOKE <权限> [, 权限] …

ON <对象类型> <对象名> [, <对象类型> <对象名>]

FROM <用户> [, 用户] … [CASCADE | RESTRICT]

3.2.3 存取控制机制

常用方法

  • 强制存取机制（MAC）发生在DAC之后
    • 主体和客体
    • 敏感度
    • 只能修改（写）同级或者更高级（此时数据未泄漏）的相关数据

3.2.4 审计

在SQL语言之中，用audit，noaudit

3.2.5 数据加密

3.3完整性控制

3.3.1 包括的几个方面

3.3.2 域完整性

not null

unique

default

3.3.3 参照完整性

某些操作会破坏完整性

    create assertion 断言名

    drop assertion 断言名

    触发事件（insert, delete, update, 或者前三个的组合）

    按间隔划分

    1. 行级触发器

    2. 语句级触发器

    rollback（理解为撤销）
