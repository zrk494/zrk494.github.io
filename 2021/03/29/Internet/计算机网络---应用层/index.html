1.2 进程通信（进程的定义参照操作系统）

1.2.2 套接字（socket，又名编程接口）

应用层和运输层之间（进程与计算机网络之间）的接口，在创建一个因特网应用之后，要选择一个运输层协议（UDP或TCP）。

1.3 应用层协议

1.3.1 内容

  • 交换的报文类型（请求还是响应）
  • 报文类型的语法
  • 字段的语义
  • 进程什么时侯发送报文
  • 进程怎么发送报文

  • 吞吐量
  • 定时
  • 安全性

2. HTTP（超文本传输协议，定义了报文格式和交换报文的方式）

2.1 相关概念解释

2.1.1 web页

若干对象组成的一个界面

2.1.2 对象

像是HTML，JPEG，Java程序之类的文件，地址用URL来表示

2.1.3 URL

统一资源定位符，组成包括两部分，主机名+路径名

2.1.4 浏览器

web应用的用户代理（客户机端）

2.1.5 web服务器

存储微博对象的地方（服务器端）

2.1.6 HTTP的工作

定义了Web客户向web服务器请求web页面的方式，以及服务器向客户传送web页面的方式

2.2 HTTP的工作过程

建立TCP连接（通过套接字接口访问TCP）—->交换报文—->关闭连接

补充说明：

  • 使用TCP作为运输协议
  • 服务器向客户发送被请求的文件，但不存储任何关于该客户的状态信息

2.3 HTTP连接方式

2.3.1 持续性连接

一个TCP连接只传送一个web对象

2.3.2 非持续性连接

一个TCP连接传送多个web对象

2.4 报文格式

2.4.1 HTTP请求报文

请求行（方法字段+URL字段+HTTP版本）+首部行

补充：

  • 首部包括多个内容

  • 方法字段

2.4.2 HTTP响应报文

状态行（版本+状态码+短语）+首部行+实体体

补充：

  • 状态码，与短语相对应，指示出了请求的结果
    • 200 OK
    • 301 Moved Permanently
    • 400 Bad Request
    • 404 Not Found
    • 505 HTTP Version Not Support

4. DNS（采用UDP传输协议）

4.1 DNS提供的服务

4.1.1 标识主机的两种方式

  • 主机名 如www.google.com之类的便于人记忆的名称
  • IP地址 一串数字，便于路由器处理，如192.165.153.11（自己瞎写的）

4.1.2 DNS（域名系统）

功能是实现主机名到IP地址转换的系统

是一个DNS服务器实现分布式数据库

它允许主机查询分布式数据库的应用层协议

通常被其他应用层协议使用

具体过程：

用户主机运行着NDS应用的客户端—–>浏览器解析URL之中的主机名，并发送给客户端——->客户端向DNS服务器发送请求（包括主机名）———>客户端收到含主机IP地址的回答报文———->浏览器向该IP地址的HTTP服务器发起TCP连接

4.1.3 主机名转换为IP地址之外的其他服务

  • 主机别名

  • 邮件服务器名

  • 负载分配

4.2 工作原理相关

4.2.1 不使用集中式设计

4.2.2 分布式，层次数据库

  • 三种类型的DNS服务器 根DNS服务器（最上层），顶级域DNS服务器（第二层），权威DNS服务器（最底层）

  • 上层包括了下层的地址

  • 本地DNS服务器

6. 套接字编程（在用户的机器上开发）

6.1 演示套接字编程的程序的过程

客户发送一串字符到服务器—–>服务器接收到数据之后将字符转换为大写——>服务器将修改后的数据发送给客户—–>客户接受到后在显示器上显示该行数据

6.2 UDP实现

6.2.1抽象

不需要建立连接，封装信息后生成分组再发送

6.2.2 应用程序

产生输入流，建立套接字——>输入—–>封装——->发送

6.3 TCP实现

6.3.1 抽象的过程

先建立连接（1. 创建本地套接字，指定服务器进程的地址—–IP+端口 2. 服务器接到请求之后，建立自己的套接字，之后建立TCP连接），再传输数据（通过TCP连接建立的管道，进行可靠的数据传输）

6.3.2 在应用程序中

产生输入输出流，建立两边的套接字—–>输入—–>发送——>接收—–>显示
