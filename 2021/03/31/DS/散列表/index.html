数据结构--散列表

发表于 2021-03-31 更新于 2022-12-01 分类于 数据结构&算法

散列表（hash map）

  • 注：在实际开发过程中，拉链法是更常用的冲突处理方式，但考试更常考开放地址法（乐）
  • 后续补充

相关概念

  • 散列表：一种特殊的数据结构，特征为数据元素的关键字和存储地址存在一个特定的关系（直接相关）。
  • 同义词：不同的关键字，通过散列函数得到的值相同
  • 冲突：通过散列函数得到的位置已经被占用
  • 散列函数：一个反应“特定关系“的函数，计算的值即为关键字的位置
  • 装填因子（常考）：表的记录数/散列表长度
    • 装填因子直接影响散列表的查找效率

散列函数

  • 设计散列函数，主要的目标是让不同关键字的冲突尽可能少
  • 散列函数的常见设计方法
    1. 除留余数法：即H(key)=key%p（一般p取不大于表长m的，且最接近于m的质数p）
    2. 直接地址法：即H(key)=key，H(key)=a*key+b
    3. 选关键字分布均匀的若干位作为散列地址
    4. 平方取中法
  • 散列查找的核心思路依靠“用空间换时间”，所以要尽可能设计一个时间效率高的算法，但同时也不要去浪费过多的空间

处理冲突

  1. 拉链法（链地址法）

    • 将“同义词“存储在同一位置的一个链表上，如下图

    • 效率分析

      • 查找长度：对比关键字的次数，如果该位置没有连着链表，则为0
      • 根据ASL的分析可得，冲突越多，效率越低
      • 可以通过一些方式，改善装填因子来提高效率

  2. 开放地址法

    • 就是根据一个序列$d_i$ ，将关键字的同义词按这个序列插入到表的新的位置（其中i表示第i次发生冲突）
    • 根据不同的$d_i$，可以具体分成3种方法
      1. 线性探测法$d_i=d_{i-1}+1,d_0=0$，即一次往后移动一格，直到用空位将这个关键字插入
      2. 平方探测法，$d_i=0,-1,1,-2^2,2^2……-k^2,k^2$
      3. 伪随机序列法，不是很重要
    • 在这个处理冲突的方法中，如果删除某个元素，不能简单的将其所在位置设置为空，而是要将此位置标记为已删除，因为失败查找的结束条件是此位置为空
