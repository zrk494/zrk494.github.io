计算机网络---数据链路层

发表于 2021-05-11 更新于 2022-11-28

数据链路层

概述

  • 基本概念

    • 两种类型的网络链路
      • 点对点链路 一对一
      • 广播链路 链路被多个设备共享
    • 结点：指主机和路由器等设备
    • 链路：连接相邻节点之间的物理信道
    • 数据链路：连接相邻节点之间的逻辑信道
    • 帧：链路层的协议数据单元，封装网络层的数据报得到

  • 数据链路层的工作就是通过一条数据链路，从一个节点向另一个与其直接相连的节点传送数据报（传输之前将数据表封装成帧）

  • 透明传输：保证传输数据的比特组合能被识别出来，不会接收方误认为是控制信息

    • 实现透明传输靠4种不同的组帧方法

  • 数据链路层的功能

    • 成帧（组帧）：即将数据报封入数据帧（添加了头和尾）

      • 帧同步：接收方从收到的二进制比特流中区分处帧的起始和终止（头和尾）
      • 组帧的4种方法
      1. 字符计数法：帧头部有一个字节的字段，记录帧内的字符数
      2. 字符填充法
      3. 零比特填充法：先在帧的头部和尾部都放一个字节（即01111110），在发送端，会扫描整个信息字段，如果扫描到连续的5个1，则在这之后立即填入一个0，在接收端，先找到边界，再扫描比特流，如果出现连续的5个1，则删除这之后的0
      4. 违规编码法：举个栗子，对于曼彻斯特编码，用“高-高”和“低低”这样的信号来界定帧的起始和终止
      • 注：前两种方法存在比较明显的问题，实际上并不常用，3和4都实现了透明传输，不会误判帧的边界

    • 链路管理，即连接的建立、维持和释放

    • 为网络层提供服务（包括无确认无连接服务/有确认无连接服务/有确认面向连接服务）

    • 流量控制

    • 差错检测和纠正（差错包括帧错和位错误）

差错检测和纠正

  • 差错产生的原因：有噪声引起

  • 两类差错

    1. 位错误：比特流的某一位错误，1变成0或者0变成1
    2. 帧错误：帧丢失/帧重复/帧失序（一般解决方法就是丢弃+重传）

  • 检错编码1——奇偶校验码

    • 奇校验码：在附加一个校验元（即在信息编码的尾部加一个比特），保证1的个数为奇数
    • 偶校验码：在附加一个校验元，保证1的个数为偶数

  • 检错编码2——循环冗余码

    • 生成多项式：双方约定好的一串序列（一般题目中有），用于辅助计算是否出错，这个多项式有r位
    • 对于发送端：求出要传输的数据（数据有d位，还会在尾部补r个0，这个最终结果用于计算）和生成多项式作商的结果和余数，将最后的r位换成余数，发给接收端
    • 对于接收端：接收到的数据与生成多项式作商，余数为0，则判断为未发生错误
    • 这个过程中，如果判断某个帧有错误，则将这一帧丢弃

  • 纠错编码——海明码（可以发现错误，找到其位置，并纠正—-0换为1，1换为0）

    • 海明距离（码距）：两个合法编码的对应位比特取值不同的比特数
      • 举个栗子：000,100他们俩的海明距离为1（只有第1位不同）
    • 流程如下
    1. 确定校验码位数r
      • 满足不等式$2^r\ge m+r+1$，m表示数据的位数，根据不等式找r的最小值

    2. 确认校验码和数据的位置

      • 校验码放在$2^n$的位置（n为位数，最右边为1）
      • 数据依次放入空位即可

    3. 求出校验码的值

      • 根据序号的二进制形式判断每一位校验码对应的检验位置

    4. 检错并纠错

      • 根据每一位校验码对负责的比特位得到一个序列，根据偶校验的方式判断这个序列是否有误
      • 满足偶校验则记为1，不满足则记为0，有高位向低位排列每一位校验码对应的结果，序列表明了第几位有错

可靠传输机制+流量控制

  • 可靠传输：保证发送方发出的内容与接收方收到的内容一致
  • 流量控制：在接收方的缓存空间有限，通过控制发送的速率保证接收方有足够的空间接收传来的帧（传输层和数据链路层都有这个机制）
  • 3种流量控制的协议（也同时保证了可靠传输）

  1. 停止等待协议

    • 目的是解决丢包（由多种原因导致数据包丢失）问题

    • 为了讨论方便，仅考虑一方发生一方接收，并且更多关注传输的机制不用考虑数据在哪一个层次上传输

    • 停止等待：发送方发送完一个分组之后就停止发送，直到收到接收方的确认信息再发送下一个分组

    • 在这个协议中的几种传输情况

      • 无差错：发送一个分组，等接收方传回一个ACK（确认帧）之后开始传下一个分组

      • 有差错情况A：分组丢失或者检测到分组出错：会设置定时器，发生一个分组就启动一个定时器，如果分组丢失则等到超时就再次发送该数据分组

        注1：发送完一个数据分组，会保留其副本

        注2：数据分组和ACK都有编号

      • 有差错情况B：ACK丢失

      • 有差错情况C：ACK迟到

    • 性能分析：信道利用率太低

  • 注1：对于这3种协议，重点在具体控制的模式，属于哪一层的不必在意
  1. 后退N帧协议（GBN）：在停止等待协议之上就行改进
    • 通过流水线技术改进（连续发送多个数据分组）
    • 发送窗口：规定了发送方一组连续发送的数据分组的序号范围（比如从第0个数据分组到第5个数据分组）
    • 接收窗口：规定了接收方一组连续接收的数据分组的序号范围
    • 接收ACK：收到ACK之后，会将该ACK对应的数据分组标记为已接收，并且会将发送窗口移动到这个分组之后
    • 累计确认：如果发送方收到n号的ACK分组，则表明n号之前的分组已经全部被接收
    • 超时处理：发送方重传丢失分组以及之后的已发送但是未被确认的分组（某一个分组出错也类似于这个过程）
    • 发送窗口最大为$2^n-1$，接收窗口大小为1
    • 存在的问题，累计确认的模式可能会丢失大量的分组，导致传输效率降低
  • 注2：这个过程结合图理解
  1. 选择重传协议
    • 在GBN协议上进行改进，设置单个确认+增大接收窗口+设置接收缓存（缓存乱序到达的帧）
    • 收到ACK：开始同GBN，但是发送窗口移动到此时最小序号的未确认的分组处
    • 超时事件：对应发送方，超时发生的时候只会重传一个数据分组
    • 对于接收方，接受窗口的范围内的分组“来者不拒”（失序的帧会被缓存），窗口会移到最小序号的未收到的分组处
    • 对应接收方，发送来的分组序号在窗口左边或者右边，则直接丢弃，左边的则还会发送一个ACK给发送方
    • 两边窗口最大为$2^{n-1}$
  • 注3：这个过程同样结合图理解

信道划分介质控制

3.1 概述
  • 两种链路：
    1. 点对点链路
    2. 广播式链路
  • 存在的问题：在共享信道上，两个结点之间的通信可能受到其他节点的干扰，导致冲突（冲突指结点同时收到两个或者多个来自其他结点的信号，无法从信号中得到有效的信息）

  • 解决方法：（介质访问控制）解决了哪个设备使用共享链路的问题，尽可能的避免干扰（如何协调多个发送和接收节点对一个共享信道的访问）

  • 信道划分方式包括动态的和静态的方式（详细内容在下一点）

  • 节点通过多路访问协议规范它们在共享信道上的传输

  • 基于原本的信道进行协调
  • 多路复用技术：多个计算机或者终端设备共享信道资源

介质访问控制

  • 信道划分介质访问控制（多路复用）——-静态分配信道的方式，提前给用户分好的信道

    • TDM（时分多路复用）：用户交替使用信道（类似操作系统的并发过程）

      • STDM（时分多路复用改进）：提高信道利用率，通过集中器收集多个用户通信的信息，动态分配时隙，在一条线路上将信息发出

    • FDM（频分多路复用）：划分出多个不同频率给不同用户（用户占用不同带宽）

    • CDMA（码分多路复用）：不同序列相互正交

      • 过程：到公共信道上线性相加，之后再分类

    • WDM（波分多路复用）：按照光的波长划分

  • 随机接入协议（一种动态分配信道的方式）——-用户可以随机发信息，发送信息时占用全部带宽
    • 时隙ALOHA（最简单的）
      • 所有帧大小相同
      • 时间被划分成等长的时隙（一个时隙传一个帧）
      • 只能在时隙开始发送帧
      • 结点之间时钟同步
    • 过程
      1. 结点有一个新的帧就在下一个时隙发送
      2. 无冲突就继续发
      3. 有冲突就在下一个时隙按概率p发送没发送成功的帧，直到成功发送
    • ALOHA（想发就发）
      • 不监听信道，不划分时隙，不需要同步，有帧生成就立刻发送
      • 效率更低
    • CSMA（载波侦听多路访问协议）
      • 特征
        • 要监听信道
        • 当监测到信道空闲时发送帧，信道忙就等待（推迟发送也存在多种方式）
        • 冲突仍然可能发生
      • 分类：
        • 坚持CSMA
        • 非坚持CSMA
        • p坚持CSMA
    • CSMA/CD（具有碰撞检测的载波侦听多路访问协议）
      • 对CSMA改进，会检测是否发生冲突，有冲突则通过截断二进制数退避算法等待一段时间再重发
      • 最小帧长：
        • 原理：保证帧传输时延不低于信号在总线传播时延的2倍
        • 最小帧长=总线传播时延$\times$数据传输速率$\times$2
      • 主要应用于有线局域网
      • 以太网的最短帧长为64B
    • CSMA/CA（）
      • 主要应用于无线局域网
  • 轮流协议（了解即可）
