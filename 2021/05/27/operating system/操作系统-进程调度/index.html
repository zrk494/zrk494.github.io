进程调度

1. 基本概念

  • CPU调度==进程调度（大概）
  • 进程可以控制CPU
  • 在多道程序的系统之中，将多个进程载入内存，一个进程等待I/O时，操作系统将代替该进程接管CPU的控制，并将CPU交给另一个进程，CPU调度就是解决怎样合理的将进程交给CPU运行
  • 进程的执行过程包括CPU执行和I/O等待这两个状态，在进程执行的过程之中，它会在这两个状态之间来回切换
  • CPU调度程序：是一种短期调度程序
  • dispatcher：将CPU的控制权交给CPU调度（还要实现上下文切换等功能）
  • 发生CPU调度的时候
    • 进程从运行状态到等待状态（可能进程发出了I/O请求）—–抢占的调度
    • 进程从运行状态到就绪状态（可能发生I/O中断）—–非抢占的调度
    • 进程从等待状态到就绪状态（可能I/O完成了）—–非抢占的调度
    • 进程终止—–抢占的调度
  • 非抢占和抢占的区别
    • 非抢占的调度是在整个过程之中进程会一直占用CPU，直到进程结束，或者进程进入等待状态
    • 抢占的调度是在进程在使用CPU的时候会被其他进程抢占使用权

2. 调度准则

  • 进行CPU调度是要参考哪些指标：
    • CPU使用率
    • 吞吐量
    • 周转时间
    • 等待时间：在就绪队列中等待的时间
    • 响应时间：从提交请求到产生第一次响应的时间
  • 等待时间和响应时间是描述调度算法最重要的两个指标

3. 调度算法

注：

  • 实际的调度过程非常复杂，下面讨论的算法中每个CPU仅被一个进程占用，大部分情况下的比较指标是平均等待时间
  • 引入甘特图直观展示调度算法，数字指时间

3.1 先到先服务调度（FCFS或者FIFO）

  • 先向CPU发送请求的进程先使用CPU，不会发生抢占

  • 存在一种问题：当一个会长时间占用CPU的进程先发送了请求，会大幅度提高平均等待时间

  • 实例 1. 2.

    1的平均等待时间为（3+6+0）/3=3

    2的平均等待时间为（24+27+0）/3=17

3.2 最短作业优先调度

  • 选择占用CPU时间最短的进程（在就绪队列之中比较进程占用CPU的时间），当两个进程占用CPU的时间相同时按先到先服务的原则

  • 在平均等待时间层面看，最短作业优先调度算法是最优的算法，能解决上面描述的问题

  • 存在的问题：预先知道下一个进程的CPU占用时间是比较困难的

  • 两种情况：抢占式和非抢占式

3.3 优先级调度

  • 将进程设置一个优先级数字，数字小的优先级高，在就绪队列之中比较进程的优先级

  • 两种类型

    • 抢占式优先级调度

    • 非抢占式优先级调度

  • 存在的问题：优先级低的进程可能永远无法运行

  • 解决方法：在较长的一段时间中，如果优先级较低的进程未被调用，则提高该进程优先级

3.4 轮转发调度

  • 是FCFS的变种，给进程增加了时间片（一般为10ms到100ms），系统每隔一个时间片就会调度另一个进程（通过抢占来实现），原本的进程被放入就绪队列（此时的就绪队列是循环队列）

  • 如果进程在在时间片内结束，则立即调用下一个进程

3.5 多级队列调度

  • 就绪队列被分成多个，每个队列采用自己的调度算法，还存在就绪队列之间的调度机制
  • 队列之间的调度
    • 固定优先级抢占调度
    • 队列之间划分时间片

3.6 多级反馈队列调度

  • 此种情况下，一个进程可以被分配到其他的就绪队列之中，通过这样的方式可以防止产生永远不会被调度的进程的产生

4. 多线程调度

4.1 概述

  • 指如何调度线程，而在实际情况中，内核并不能直接调度用户线程（是由线程库管理的），用户线程要先映射到内核线程（建立联系，参照上一章的那几个模型），再通过调度内核线程来完成用户线程的间接调度

4.2 两种调度方式

  • Process-Contention Scope（PCS）—–用户进程间的调度（只存在多对一，多对多模型之中）
    • 指线程库调度用户线程到LWP之中
    • 此过程根据优先级完成，优先级是人为设置的
  • System-Contention Scope（SCS）
    • 系统将内核线程调度到CPU上

5. 多处理器调度

5.1 概述

  • 系统中有多个处理器，多处理器调度规定多个CPU如何共同工作完成进程调度

5.2 两种可行的调度方法

  • 非对称处理：选择一个CPU作为主处理器，处理调度相关的事务，其余的处理器听从其指挥，只执行用户的代码
  • 对称处理：每个处理器使用自己的调度算法，从一个公共的就绪队列之中选择进程，调度过程之中必须确保处理器不会选择同一个进程
