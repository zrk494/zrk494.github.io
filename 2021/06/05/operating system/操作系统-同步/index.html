操作系统---同步

发表于 2021-06-05 更新于 2022-10-06

进程同步

概述

同步机制
  • 操作系统通过同步机制，保证指令按可以预计的情况去依次执行，并且不产生错误或者卡在某条指令无法执行
临界区问题
  • 临界资源：一个时间段内只允许一个进程使用的资源（各进程对其的访问是互斥的）
  • 使用临界资源一般是与I/O相关的操作
  • 内核程序临界区和一般临界区的区别
    • 内核的临界区一般用来访问内核数据结构，访问时会上锁，不能发生调度和切换
    • 而非内核中的临界区可以（比如请求打印机）
进程互斥

  • 访问临界资源的时候，必须互斥进行，即一个进程访问临界资源是，另一个进程要访问该进程就要等待，直到访问临界资源的进程结束访问。

  • 互斥访问临界区，要依次实现4个部分的代码逻辑

    1. 进入区：检查进程是否可以进入临界区（如果可以进入，则会给该进程设置标识，表明其正在访问临界区）
    2. 临界区（临界段）：访问临界区的代码
    3. 退出区：解除之前设置的标识
    4. 剩余区（不重要）
    • 注：互斥访问主要在进入区和退出区实现

  • 为了实现互斥访问临界资源，要遵循以下原则

    1. 空闲让进
    2. 忙则等待
    3. 有限等待
    4. 让权等待

进程互斥的硬件实现方法

中断屏蔽
  • 通过中断屏蔽保证进程访问临界区时不允许被中断
  • 简单高效，但不适合多处理机，只能在内核进程使用，不能在用户进程使用
指令test_and_set()（TS指令/TSL指令）

  • 通过代码描述指令逻辑（实际是由硬件实现的）

  • 实现简单，使用多处理机

  • 不满足让权等待（导致忙等待）
指令compare_and_swap()（Swap指令/Exchange指令）

  • 通过代码描述指令逻辑（实际是由硬件实现的）

互斥锁

  • 互斥锁是指一种实现进程互斥的思路，具体实现可以通过硬件也可以通过软件
  • 需要连续循环忙等的互斥锁，都可以称为自旋锁（比如之前所述的几种方式）
  • 不适用于单处理机系统

信号量

概述
  • 通过引入信号量实现让权等待

    • wait可以简写为P(S)，signal可以简写为V(S)

信号量的具体应用

实现进程互斥
  1. 划分临界区
  2. 设置互斥信号量处置（mutex=1）
  3. 在进入区完成P操作，申请资源
  4. 在进入区完成V操作，释放资源
  • 注1：不同的临界资源使用不同的信号量
  • 注2：P操作和V操作都是成对出现在同一进程
实现进程同步
  1. 确定在哪些地方要实现“同步关系”（哪些操作要保证“一前一后”的顺序执行）
  2. 设置同步信号量为S，初始化为0；
  3. 在前操作之后执行V(S)，释放资源
  4. 在后操作之前执行P(S)，申请资源
  • 注1：这个过程本质上是实现一对进程的同步（一个进程同步问题）
  • 注2：P操作和V操作都是成对出现在不同进程
  • 复杂关系的同步
    • 前驱，即前进程
    • 一对前驱关系对应了一个进程同步问题
    • 先得到语句的前驱图，根据图得到结果
  • 简单总结：
    1. 互斥关系：在临界区前后分别PV（一个进程内部）
    2. 同步关系：前V后P（对应前后两个进程）
    3. 实现互斥的P必须在实现同步的P之后。否则会死锁

问题条件

  1. 系统中有一组生产者进程+一组消费者进程，生产者每次产生一个产品放入缓冲区，消费者每次从缓冲区中取出一个产品使用

  2. 设缓冲区大小为n，初始为0

  3. 缓冲区满时生产者不能继续放入产品
  4. 缓冲区空时消费者不能取走产品

多生产者多消费者问题

问题描述
  • 存在一个大小为1的缓冲区，以及2个（或者以上）的生产者+2个（或者以上）的消费者，一对生产者消费者对应一种产品
问题分析
  • 关系分析
    • 互斥关系：互斥的访问缓冲区
    • 同步关系（一前一后）：
      1. 生产者将产品放到缓冲区，之后其对应的消费者才能拿走产品（有几对生产者和消费者，就有几个这样的同步关系）

哲学家进餐问题

管程

诞生背景
  • 信号量存在问题，编写困难且易出错，我们希望存在一种机制，不再关注复杂的PV操作，让写代码更轻松
  • 管程简单来说就是一种解决上述问题的高级同步机制
定义+基本特征
  • 包括的功能（相当于一个类）
    1. 共享数据结构说明
    2. 对该数据结构的一组操作
    3. 对共享数据的初始化
    4. 管程的名字

  • 特征：

    1. 内部数据只能通过内部的方法访问
    2. 一个进程只能调用管程的方法才能访问共享数据
    3. 每次只有一个进程能执行管程的内部方法
  • 由编译器实现进程互斥地进入管程

  • 可以在管程中设置条件变量和等待唤醒操作解决同步问题

  • 注：管程本质其实就是面向对象的模式设计了一个管程类
