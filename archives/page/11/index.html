操作系统---概述

发表于 2021-03-07 更新于 2022-08-08

操作系统概述

操作系统的相关基本概念

  • 操作系统，是包括以下几个功能的软件系统

    1. 控制和管理计算机系统的软件硬件
    2. 组织和调度计算机的工作和资源分配（核心目的是提高CPU的利用率）
    3. 为用户和软件提供方便的接口与环境

  • 操作系统的4个重要的特征

    1. 并发（Concurrence）：两个或者多个事件在同一时间间隔发生（在微观上是分时交替进行），注意与并行的区别
      • 注，这个过程结合例子理解：9:10-9:20只吃东西，9:20-9:30只写字，9:30-9:40只吃东西，9:40-9:50只写字，这个过程就是并发（9:10-9:30吃东西，同时写字，这个过程是并行）
    2. 共享（Sharing）：系统中的资源可以由并发的几个进程共同使用（但是在一段时间内，只能有一个进程访问这些资源）0注，共享是以并发为条件，没有并发的机制，就不会存在共享，同样，如果不能通过共享来有效的管理资源，则会影响到并发的过程
    3. 虚拟（Virtual）：虚拟是指把物理上的实体变为若干逻辑上的对应物（实际存在比用户感觉要少），根据后续章节慢慢理解
    4. 异步（Asynchronism）：进程在指向过程中不是一贯到底的，总会有执行和等待的状态交替进行，这样的特性就是异步性。
    • 注：1,2是操作系统最基本的特征（3,4的实现基于1和2）

  • 操作系统为用户提供的交互方式

    1. 命令接口（允许用户直接使用，shell）

      1.1 联机命令接口（交互式命令接口），就比如windows下的cmd

      1.2 脱机命令接口（批处理命令接口），比如.bat文件

    2. 程序接口（考试中可以将其等同于系统调用，但实际上存在一些差别）：有一组系统调用组成的接口，用户可以间接调用

  • 补充1：命令解释器，解释如用户在cmd中输入的命令

  • 补充2：BIOS(basic input output system)

    • 在裸机上，BIOS系统存在ROM中，安装操作系统或者开机的时候会用到

操作系统的发展阶段

1 手工操作阶段
  • 在计算机上所有过程的计算都要人为操作，等待时间长，资源利用率低
2 批处理系统

  • 单道批处理系统

    • CPU按顺序对任务进行处理，输入输出不需要人操作，交给机器按顺序一个一个作业依次完成

  • 多道批处理系统（因为输入输出的运行时间较长，所以以此方式改进单道的方式，提高CPU利用率）

    • 有多个程序并发执行，CPU处理完一个任务之后，直接开始处理下一个任务，不会等待I/O（输入输出）

3 分时操作系统

  • 批处理系统中不存在人机交互的功能，而设计分时操作系统来解决这个问题

  • 将处理器处的运行时间划分为很短的时间片，按时间片轮流将处理器分配给任务（作业）去使用，即在宏观上感觉各个用户在同时在与计算机交互

4 实时操作系统
  • 分时操作系统中，任务不区分优先级，循环地按照时间片分配处理器给用户，这样存在一些紧急任务无法立刻执行，故通过实时操作系统改进
  • 存在优先级，优先级高的先执行，某些紧急的任务不需要按时间片排队
  • 软实时操作系统
  • 硬实时操作系统

操作系统的运行

  • 在程序的运行过程中，系统发生怎样的变化
  • 两种指令（指令是处理器（CPU）能识别、执行的最基本命令，注意与交互式命令接口的区别）
    • 特权指令，一些比较危险，不能让用户去使用的指令
      • 具体例子：输入输出指令（I/O），开中断指令，关中断指令（屏蔽中断指令），修改PSW指令
    • 非特权指令，基本运算之类的
  • 两种处理器状态（或者描述为操作系统的状态）
    • 核心态（内核态，管态）
      • 处于内核态时，说明此时正在运行的是内核程序，此时可以执行特权指令
    • 用户态（目态）
      • 处于用户态时，说明此时正在运行的是应用程序，此时只能执行非特权指令
    • CPU 中有一个寄存器叫 程序状态字寄存器（PSW），其中有个二进制位，1表示“内核态”，0表示“用户态”
    • 二者之间的切换：
      • 内核态到用户态通过一条特权指令，修改PSW的值来完成
      • 用户态到内核态只能由中断引起，硬件自动完成切换，操作系统重新活动CPU使用权。
  • 两种程序
    • 内核程序：在操作系统内核中，实现操作系统核心功能的程序，最接近硬件部分
    • 应用程序：运行在操作系统内核外的程序
  • 陷入指令（访管指令，trap指令）
    • 一条特殊的非特权指令
    • 只会在用户态下使用（因为内核没有必要使用这个指令）
    • 会将CPU控制权交还给内核（比如系统调用就是通过这个指令完成）
    • 陷入指令被用户程序调用之后，会产生一个中断事件，让操作系统的状态发生变化
中断和异常
  • 中断用于下达指令通知事件的发生，发生之后会中断应用程序执行，然后去执行内核程序
  • 内中断（也称作异常，软中断，信号来自CPU内部）
    1. 陷入，陷阱（trap）：陷入指令引发，往往是程序故意引发的
    2. 故障（fault）：由错误的条件引起，可能被内核修复，修复完会将CPU使用权归还给应用程序
    3. 终止（abort）：由致命错误引起，内核无法修复，
  • 外中断（可称直接为中断，硬中断，信号来自外部，一般与硬件相关）
    1. 时钟终端
    2. I/O中断请求
  • 中断向量表（保存着中断程序的地址和类型及其对应）
  • 中断机制基本原理（或具体过程）
    • 发生中断时，系统保存当前进程的程序计数器（PC）
    • CPU查询中断向量表，找到其位置，进行中断处理程序
    • 中断服务结束后，CPU返回到被中断的程序（通过PC找到中断的位置），程序继续向下进行
系统调用
  • 系统调用=系统调用命令=广义指令：操作系统为用户提供了其与计算机硬件之间交互的接口，接口向用户提供了简单易用的服务，包括程序接口和命令接口，而系统调用就组成了这里的程序接口（也许就是操作系统提供的一组特殊命令（函数），可由应用程序调用）
  • 系统调用发生在用户态，之后的响应操作发生在内核态

  • 与库函数比较

    • 库函数可能对系统调用做了一次封装，隐藏了细节，使调用更方便
    • 系统调用可以直接用程序调用，也可以通过库函数触发
    • 库函数不一定是操作系统提供的，而系统调用只能是操作系统提供给编程人员的

操作系统内核
  • 内核主要的4个功能
    • 时钟管理
      -
    • 中断处理
      -
    • 原语操作
      • 一种特殊的处于操作系统最底层的，最接近硬件部分的程序
      • 具有原子性，即不可被中断
    • 管理的数据结构及其处理
      • 进程管理
      • 存储器管理
      • 设备管理
    • 注：前三个功能与硬件紧密相关，而第四个更多是对数据结构进行管理，不直接涉及硬件

  • 大（宏）内核

    • 所有系统功能都放在内核里，通常内核中的程序也是模块化的设计思路

  • 微内核

  • 分层结构

    • 内核分成多层，每层会为上层提供调用接口（每一层只能调用更低一层的接口，不能跨层调用）
    • 优点
    • 缺点

  • 模块化

    • 将内核划分成多个模块，包括主模块（负责核心功能，如进程管理）和可加载内核模块（一些不影响系统运行的模块）

  • 外核

    • 内核负责，而划分出一个外核负责为用户进程分配未经过抽象（即一片连续的物理地址空间，而不是虚拟的一片连续的空间）的硬件资源，同时保证使用这批资源是安全的
    • 优点
    • 缺点
操作系统的引导
  • 就是开机的过程中，如何让操作系统在电脑上运行起来
  • 磁盘里存的内容
    • 主引导记录（MBR），存在磁盘头部，包括两部分：
      1. 磁盘引导程序
      2. 分区表（即每个盘的活动范围）
    • 活动分区（在磁盘上划分的区域，比如C盘，D盘），每个活动分区有3个部分
      1. 引导记录：负责找到启动管理器
      2. 根目录
      3. 其他
  • 补充内容：主存，主要包括两部分
    • RAM
      • 关机数据会被清空
    • ROM
      • 存有BIOS——Basic Input/Output System
      • 还包含了ROM引导程序
      • 断电后数据还在
  • 引导的主要过程（本质上就是根据引导，不断执行一系列的程序，直到操作系统启动）
    1. CPU从一个特定的主存地址开始，获取指令，执行ROM之中的引导程序（执行之前会检测一下是否存在硬件上的问题）
    2. 将磁盘的第一块，就是主引导记录读入内存，CPU执行其中的磁盘引导程序，并开始扫描分区表
    3. 根据分区表，找到操作系统本体所在的活动分区，读入分区头部的引导记录，执行这块区域的程序
    4. 在根目录下找到操作系统的初始化程序并执行（比如Windows操作系统的初始化程序就在C:\Windows\Boot中），完成“开机”。
