数据链路层

1. 概述

1.1 相关概念解释

  • 节点 指主机和路由器等设备

  • 链路 连接相邻节点的信道

    • 有线链路
    • 无线链路
    • 局域网

  • 链路层帧（帧） 指链路层之中的数据分组，用于封装网络层的数据报

  • 两种类型的网络链路

    • 点对点链路 一对一
    • 广播链路 链路被多个设备共享

  • 概括 数据链路层的工作就是通过一条链路，从一个节点向另一个与其直接相连的节点传送数据报文

1.2 提供的服务

1.2.1 成帧（组帧）

将数据报封入数据帧（添加了头和尾）

1.2.2 链路接入

某一链路可能需要被多个节点使用，链路接入问题就是哪个节点去使用链路

1.2.3 可靠交付

一般来说，对于无线链路是必要的，而对于有线链路是不必要的

2. 差错检测和纠正

3. 多路访问协议（MAC）

3.1 概述

  • 在共享广播信道上，两个及以上的结点同时传输时会互相干扰，导致冲突（冲突指结点同时收到两个或者多个来自其他结点的信号）

  • 解决了哪个设备使用共享链路的问题，尽可能的避免冲突（如何协调多个发送和接收节点对一个共享广播信道的访问）

  • 通过算法决定节点如何合理的使用共享信道（节点通过多路访问协议规范它们在共享信道上的传输）

  • 基于原本的信道进行协调

3.2 三种MAC协议

3.2.1 信道划分MAC协议（多路复用）

  • TDM（时分多路复用）
  • FDM（频分多路复用）
  • CDMA（码分多路复用）

3.2.2 随机接入协议

  • 此类协议的概述

    • 不划分信道
    • 出现冲突再恢复即可
    • 结点在信道上用全速发送分组
    • 事先结点不协调，随时可能将报文发出去
    • 两个需要定义的重要事项
      • 冲突检测机制
      • 恢复机制（发送失败了之后怎么做）

  • 具体例子（递进）

    • 时隙ALOHA（最简单的）

      • 前提假设
        • 所有帧大小相同
        • 时间被划分成等长的时隙（一个时隙传一个帧）
        • 只能在时隙开始发送帧
        • 结点之间时钟同步
      • 过程
        1. 结点有一个新的帧就在下一个时隙发送
        2. 无冲突就继续发
        3. 有冲突就在下一个时隙按概率p发送没发送成功的帧，直到成功发送
      • 效率太低

    • ALOHA

      • 特征（只描述与时隙ALOHA不同的地方）
        • 不划分时隙，不需要同步
        • 有帧生成就立刻发送
        • 效率更低

    • CSMA（载波侦听多路访问协议）

      • 特征
        • 要监听信道
        • 当监测到信道空闲时发送帧，信道忙就等待（推迟发送也存在多种方式）
        • 冲突仍然可能发生

    • CSMA/CD（具有碰撞检测的载波侦听多路访问协议）

4. ARP协议（地址解析协议）

4.1MAC地址（物理地址，LAN地址，以太网地址）

  • 作用：标识链路层帧是从哪一个接口发出的，以及标识要到达哪个物理相连的其他接口
  • 地址格式：48位（一般固化在网卡之中），16进制数表示
  • MAC是唯一的，是一种平面的地址（不像IP地址那样分层）

4.2 ARP

  • 解决如何由已知目的接口的IP推出该接口的MAC地址的问题—–通过ARP表解决这个问题（存储着IP到MAC地址的映射关系，还会设置一个TTL）

  • TTL结束时会删除这条映射关系

  • 同一个局域网内，A给B发数据时，如果B的MAC地址不在ARP表之中，将进行如下操作：

    • A广播ARP分组，包含了B的IP地址，此分组的目的MAC地址为FF:FF:FF:FF:FF:FF
    • 这个局域网之内的所有结点都会接收这个ARP查询
    • 只有当分组中的IP地址与结点IP地址匹配的结点即B会向A发出应答自己的MAC地址（这一次应答是B到Ａ的单播）
    • Ａ收到响应后记录后会将Ｂ的MAC地址与IP的映射记录在ARP表之中，并且设置一个超时时间

  • 不同局域网，A在的子网与B所在的子网通过路由器R连接，A向B发数据时，如果B的MAC地址不在ARP表之中，将进行如下操作：

    • 封装A的链路数据帧，目的MAC地址为A的默认网关的MAC地址，即将这个帧发给默认网关的接口（MAC的地址是通过ARP协议获得，过程参考上面的情况）
    • 路由器R提取出IP数据报，交给上层协议处理（此过程之中可能会使用到NAT协议，改变源IP的值）
    • 路由器R将IP数据报封装封装到新的数据链路帧之中，目的MAC地址为B的MAC地址，源MAC地址为路由器进入B子网的接口的MAC地址
    • 数据帧到达B后经过解析将B的MAC，IP再发给A

  • ARP是个即插即用的协议（一般情况下ARP表为空，使用的时候才建立，并且是结点自主创建的）
