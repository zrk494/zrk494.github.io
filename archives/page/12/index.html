计算机网络---应用层

发表于 2021-03-29 更新于 2022-09-24

应用层

应用层协议原理概述

  • 为应用程序的通信提供服务

  • 应用层协议定义内容概述

    • 交换的报文类型（请求还是响应）
    • 报文类型的语法（即报文格式）
    • 字段的语义（某个字段中信息表示什么）
    • 进程什么时侯发送报文，怎么发送报文
    • 进程对报文进行响应的规划

  • 网络应用模型

  • C/S (客户机/服务器)

    • 服务器对外提供服务（不间断，一般有大量的服务器），客户机使用服务器提供的服务，不会和其他客户机直接的通信
    • 服务器IP是固定不变的，而客户机是可变的，间歇性接入网络

  • P2P (点对点)

    • 端系统或者节点之间直接通信
    • 没用永久在线的主机，它们可以提供服务，也可以请求服务

  • C/S+P2P（混合）

HTTP（超文本传输协议）

概述

  • 万维网（World Wide Web）

  • URL：统一资源定位符，给资源（音频视频图片之类的）的唯一编号，组成包括两部分，主机名+路径名

  • 用户通过点击超链接获取资源（这些资源通过HTTP传输）
  • 万维网基于C/S架构工作
  • 万维网使用HTML，可以很方便的通过连接从一个界面跳转到另一个界面
  • HTTP定义了客户如何请求万维网文档，以及服务器如何将文档发给用户
HTTP的工作过程
  1. 用户输入URL/点击超链接
  2. 浏览器分析链接（或者URL）
  3. 浏览器向DNS请求IP解析
  4. DNS解析得到IP
  5. 浏览器（客户）与服务器建立TCP连接
  6. 用户发送HTTP请求
  7. 服务器返回HTTP响应
  8. 浏览器显示返回结果
  9. 释放连接
HTTP的连结方式
  • 持久连接：建立连接之后，可以在此次连接中发送多次HTTP请求获取资源
    • 流水式：连续发送，依次响应
    • 非流水式：停等
  • 非持久连接：建立一次TCP连接，客户只发送一次HTTP请求，只接收一次HTTP响应报文就关闭TCP连接
HTTP的一些特征

  • 服务器不存储任何关于该客户的状态信息（无状态协议）

  • 在实际工作中，一些站点希望能识别用户，故引入cookie技术存储用户的访问记录，同时标记出用户

  • HTTP本身是无连接的
报文格式

  • HTTP请求报文（ASCII书写）

    • 请求行（方法字段+URL字段+HTTP版本）+首部行（可扩展）+消息体（Entity Body）

  • HTTP响应报文

    • 状态行（版本+状态码+短语）+首部行（可扩展）+实体
      • 状态码，与短语相对应，指示出了请求的结果
        • 例：404 Not Found

电子邮件

概述
  • 电子邮件的格式：信封+内容（首部+主体）

  • 组成结构

    • 用户代理（电子邮件的客户端软件）
    • 邮件服务器：核心部分，主要完成收发邮件功能，同时告知用户发送情况
    • 协议：包括接收协议（如POP3）和发送协议（如SMTP）

  • 工作原理概述：

    • 用户通过用户代理基于SMTP协议将邮件发到邮件服务器（有一个邮件缓存）
    • 服务器通过SMTP协议将邮件发到接收方的邮件服务器
    • 接收方的邮件服务器通过POP3协议将邮件传给接收端的用户代理
SMPT协议
  • 规定了两个相互通信的SMPT协议进程如何交换数据

  • 基于TCP连接，端口25，使用C/S架构

  • 通信的3个阶段

    1. 连接建立
    2. 邮件发送
    3. 连接释放
  • 缺点：传输的内容受限
  • 改进协议：MIME（使传输内容更加丰富）
POP3协议
  • 基于TCP连接，端口110，使用C/S架构
  • 两种工作方式：
    1. 下载并保留在服务器
    2. 下载并删除
  • 改进协议：IMAP协议（让用户先看到邮件首部，打开后再看到完整内容）
基于万维网的邮件
  • 用户代理和服务器之间传输通过HTTP完成，服务器之间仍然用SMTP

DNS（Domain Name System）

DNS提供的服务(解决Internet上主机或路由器的识别问题)

  • 标识主机的两种方式

    • 主机名（域名） 如www.google.com之类的便于人记忆的名称

    • IP地址 一串数字，便于路由器处理，如192.165.153.11

  • DNS（域名系统）的地址转换服务

    • 功能是实现主机名到IP地址转换的系统（即翻译）
  • 简单过程描述
    • 用户输入域名，DNS服务器解析域名，用户得到目的主机IP，与目的主机通信
工作原理相关

  • 集中式设计的问题（不重要，了解即可）

    • 单点失败（一旦应该服务器坏了，互联网就崩了）

    • 流量问题

    • 距离问题

    • 维护性问题

  • 分布式，层次服务器

    • 三种类型的DNS服务器（总体结构是树）

      • 根域名服务器（最上层） 它知道所有的顶级域名服务器的位置（ip）
      • 顶级域名服务器（第二层） 负责com, org, net, edu等顶级域名和国家级顶级域名（cn，uk等）
      • 权威域名服务器（最底层） 组织内部的域名解析服务器
    • 按这个层次划分的域名系统（即树形结构）—与服务器的结构对应
      • 根
      • 顶级域名
      • 二级域名

  • 客户要查询IP时的过程（例，www.amazon.com）

    • 查询根服务器，找com域名（客户端第一次查询
    • 到第二层的com域名解析服务器，找amazon.com（客户端第二次查询

    • 到第三层amazon.com域名解析器，获取IP地址（客户端第三次查询

    • 上层包括了下层的地址，用来告诉客户端下一次查询去那里

  • 本地DNS服务器

    • 一个主机会首先向其发出DNS查询请求，
    • 如果找不到则本地DNS服务器代替主机从根域名服务器开始，逐层向下找（树结构的查找）
    • 代理作用
DNS的具体过程
  • 迭代查询（靠自己）

  • 递归查询（靠别人）

文件传输协议（FTP）

概述
  • 不同主机系统之间传输文件
  • FPT基于C/S架构
  • 提供服务的是FTP服务器，客户连接该服务器按协议传输文件
工作原理
  • 登录：一般来说需要用户名和密码，实际中很大一部分FTP服务器是匿名登录的
  • FTP协议基于TCP实现可靠传输
  • 服务器中的主进程（1个，负责接收新的请求）和从属进程（n个）

  • 具体工作过程（传输的过程）

    • 会有两个从属进程为用户提供服务，即建立了两条TCP连接
    • 控制连接：传输控制信息，整个过程一直打开
    • 数据连接：进行文件传输

  • FTP的传输模式

    • 文本传输模式
    • 二进制模式
