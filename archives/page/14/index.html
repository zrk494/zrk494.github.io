关系数据库

1.关系

1.1 相关概念

1.1.1 属性
  • 关系表的列
  • 列是同质的，即为同一数据类型
1.1.2 域

相同类型数据的集合

1.1.3 关系

笛卡尔集的子集（详见集合论与图论）

1.1.4 关系模式

关系的形
记作 R（U）,U为A1…An组成的属性集合，R是关系名
例：Student（学号，姓名，性别，年龄，专业）
Grade（学生编号，课程编号，成绩）

1.1.5 关系实例

一个关系的任何一个“当前值”是该关系的实例

1.1.6 元组

笛卡尔积得到的集合中的每一个元素

1.1.7 码（又称作键， 那个名为篝酱的少女）
超码

可以==唯一==标识一个元组的一个属性集合

候选码

唯一的一个属性集合或者属性集合组，超码是候选码的特殊表现，可存在多个

主码

数据库设计者选择的用于区分不同元组的候选码（从候选码之中选出一个）

外码

非关系R的候选码，而是关系S的候选码，将两个关系联系在一起

1.1.8 关系的第一范式

属性不可再分

1.2 关系的完整性

1.2.1 实体完整性约束

主键属性不空（不存在或者无意义）

1.2.2 参照完整性约束（连接两个关系）

外码在R1之中的值为空或者对应其在R2（在R2之中为主码）之中的值

1.2.3 用户定义的完整性

2.2 数据定义

2.2.1 create（创建，包括表，视图—一种虚拟的表，索引）

    实例：

  • create index(为加快查询而定义的)

    语法格式：CREATE [UNIQUE] INDEX 〈索引名〉 ON 〈关系名〉(〈列名 〉[ORDER], …,〈列名〉[ORDER]) [CLUSTER]

3.数据库安全性，完整性（用SQL实现）

3.1 基本概念

3.1.1安全性
  • 不允许恶意修改或者未经授权的存取和修改
3.1.2完整性（为保障数据正确性，相容性，添加的约束条件和规则）
  • 语义完整性（重点）
  • 并发控制，安全控制，DB回复等

3.2 控制安全性

3.2.1授权

语法格式

GRANT <权限> [, 权限] …

ON <对象类型> <对象名> [, <对象类型> <对象名> ] …

TO <用户> [,用户] … [WITH GRANT OPTION];

例

Grant select

on table game

to krzkrz;

3.2.2 权限回收

REVOKE <权限> [, 权限] …

ON <对象类型> <对象名> [, <对象类型> <对象名>]

FROM <用户> [, 用户] … [CASCADE | RESTRICT]
