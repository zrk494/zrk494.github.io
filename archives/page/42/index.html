计算机系统概述

发表于 2021-03-07 更新于 2022-11-28

计算机系统概述

计算机组成原理

  • 计组主要是研究计算机的硬件相关的内容

计算机硬件的结构

冯诺依曼结构

  • 核心思路是将指令以二进制代码的形式存入计算机的主存，之后计算机读取主存完成对应的一系列操作

  • 5大部分
    1. 输入设备
    2. 输出设备
    3. 存储器（存放数据和程序）
    4. 运算器（进行逻辑和算数运算）
    5. 控制器（指挥程序运行）
  • 5个特征
    1. 指令和数据以同等的地位存放在存储器之中，可以通过地址寻访
    2. 指令和数据都用二进制数表示
    3. 指令包括两部分：操作码（执行怎样的操作）+地址码（要计算的数据所在地址）
    4. 会将程序（一系列的指令）存储在存储器中
    5. 以运算器为核心
  • 注1：计算机系统中，软件和硬件在逻辑上等效（即完成一个操作可以由软件控制实现，也可以有硬件控制实现）
现代计算机结构

  • 在冯诺依曼的结构上做了一些改进，以存储器为核心

  • 现代计算机的硬件主要包括两大部分

    1. 主机
    • 包括CPU（运算器+控制器）+主存
    1. I/O设备（外设）
    • 包括输入设备+输出设备+辅存（磁盘）

计算机相关硬件

主存（3个部分）
  1. 存储体：相当于一个仓库，存储着一系列的指令
  2. 存储地址寄存器（MAR）：暂时放着要访问的
  3. 存储数据寄存器（MDR）：暂时放着待处理的数据

  • 概念补充

    • 存储单元：存放一串固定长度的二进制代码的（有一个唯一标识的地址）
    • 存储字：一个存储单元中二进制代码的组合
    • 存储字长：存储单元中二进制代码的位数
    • 存储元，存储二进制的电子元件（每一个元件存1bit）

  • MDR的位数=存储字长

  • MAR的位数表示了存储单元的个数
运算器（4个部分）
  1. ACC：累加器（存放操作数/运算结果）
  2. MQ：乘商寄存器（存储乘法和除法运算的操作数/运算结果）
  3. X：通用操作数寄存器（存放操作数）
  4. ALU：算术逻辑单元（实现算数运算+逻辑运算）
控制器（3个部分）
  1. CU：控制单元（实现控制器的分析，控制功能）
  2. IR：指令寄存器（暂时存放当前执行的指令）
  3. PC：程序计数器（存放下一条指令地址，会自动+1）

  • 补充：如何完成一条指令

    1. 取指令（根据PC找到这条指令）
    2. 分析指令（指令放在IR）
    3. 执行指令（CU中完成）
    • 1+2合称为取指，3称为执行
计算机如何通过这些硬件工作

计算机系统的层次结构

基于指令运行的划分层次（自底向上）：
  1. 微程序机器（执行微指令）
  2. 传统机器（执行机器语言指令，一串01组成的字符串）
  3. 操作系统机器（约等于接口，向上提供广义指令（系统调用））
  4. 汇编语言机器（执行汇编语言）
  5. 高级语言机器（执行高级语言）
翻译程序（将其他计算机语言“翻译”成机器指令）
  • 汇编程序（汇编语言翻译成机器指令）
  • 编译程序（只翻译一次）
  • 解释程序（每次执行都要翻译）

计算机的性能指标

主存容量
  • 总容量可以根据MAR（说明有多少个存储单元）和MDR（表示每个存储单元的大小，即存储字长）得到
  • 总容量=存储单元个数$\times$存储字长（bit）=存储单元个数$\times$存储字长/8 (Byte)
CPU的性能指标
  • CPU主频：CPU内部数字脉冲信号的频率，即时钟周期的倒数，单位赫兹
  • CPU时钟周期：单位微妙/纳秒
  • CPI：执行一条指令所需的时钟周期数
  • 执行一条指令消耗的时间：CPI$\times$CPU时钟周期
  • CPU执行时间（整个程序耗时）：CPU时钟周期的数量/主频or（指令条数*CPI）/主频
  • IPS：每秒执行多少条指令
    • KLPS（千次）
    • MIPS（百万次）
  • FLOPS：每秒进行多少次浮点运算
系统整体的性能指标
  • 数据通路带宽：数据总线一次位数
  • 吞吐量：系统单位时间内处理请求的数量
  • 响应时间：用户发送请求之后，到系统响应结果的等待时间

补充：动态测试，通过基准程序测量计算机处理速度

补充：一般机器字长就是CPU寄存器的位数
