发表于 2021-05-27 更新于 2021-06-05

线程（thread）

1. 概述

1.1 线程的定义

  • 比进程更小的运行单位（一般在同一个进程内），比进程节省空间（复制进程的其中一部分，剩下的一部分是可以共享的）
  • 每个线程是一个独立调度的对象，一个进程可以拥有多个线程（这些线程共享内存空间）

1.2 多线程

  • 一个进程中的多个线程来同时完成多个不同的处理请求
  • 有这些优点
    • 响应速度快
    • 资源可共享
    • 开销小
    • 可以更好的发挥多处理器的特性

1.3 用户线程

  • 由用户层的线程库来进行管理的线程

1.4 内核线程

  • 由内核进行维护，管理，调度的线程

2. 多线程模型

  • 描述用户线程和内核线程之间的映射关系的一种模型
  • 在模型之中，用户线程基于内核线程运行（其中一个发生阻塞则另一个也阻塞）
  • 通过这样的关系描述，便于后续进行线程调度的描述

2.1 多对一模型

  • 多个用户进程对应一个内核进程

2.2 一对一模型

  • 一个用户进程对应一个内核进程

2.3 多对多模型

  • 多个用户进程对应多个内核进程

2.4 混合模型

3. 线程库

3.1 定义

  • 为程序员提供创建和管理线程的API

3.2 两种实现方法

  • 在用户空间中提供这个库（无需内核支持，代码之类的内容都存在于用户空间中）

  • 将库嵌入到内核（代码之类的都存放在内核）

3.3 三种主要线程库（都是用户进程）

3.3.1 POSIX Pthread

3.3.2 windows

3.3.3 java

4.1 系统调用fork ()和exec ()

4.2 信号处理

  • 操作系统中内核与进程之间的通信机制，告诉进程某一事件的发生
  • 两种信号
    • 同步信号：进程本身事件产生的信号
    • 异步信号：进程之外事件产生的信号
  • 处理过程
    1. 特定的事件产生一个信号
    2. 信号被传给进程
    3. 进程中的信号处理程序处理信号（立刻处理）
  • 注意事项

    • 在多线程进程之中，信号应当具体到传给哪一个进程，存在以下的几种情况

4.3 线程撤销（取消）

  • 定义，在某一线程完成之前将其终止，该线程称作目标线程
  • 两种取消方式
    • 异步取消：一个线程终止目标线程
    • 延迟取消：目标线程检查自己是否可以终止，在一些情况下可以将自己终止

4.4 调度程序激活

  • 是一种用户进程库和内核之间通信的方法
  • 在多对多模型之中使用调度程序激活以保持适当数量的内核线程
  • 引入了一个数据结构（LWP）放在用户和内核之间
    • LWP对用户来说是一个虚拟处理器
    • 一个LWP连接着一个内核线程
    • 内核阻塞，则LWP也阻塞，连接着LWP的用户线程也阻塞
  • 激活的过程
    • 内核提供一组LWP给应用程序，应用程序

4.5 线程地

  • 在进程开始的时候创建一定数量的线程，放在线程池中等待工作
    • 在线程池之中的线程被直接调用，减去了创建的时间
    • 限制了可用线程的数量，便于管理
