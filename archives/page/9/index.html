面向对象基础

发表于 2021-06-26 更新于 2021-06-30

建立对象的四个基本原则

1. Abstraction（抽象）

  • 将对象（现实中的实体）进行抽象
  • 算法抽象+数据抽象
  • 抽象的原则

2. Encapsulation （封装）

  • 抽象的下一步工作
  • 封装是对信息进行隐藏，隐藏了对象的关键信息（具体见实例）
  • 为什么要封装
  • 接口与封装

3. Modularity （模块化）

  • 模块化将抽象包装成不同的离散的单元

4. Hierarchy 层次化

  • 利用继承构造层次
  • 聚合关系

类

1.定义相关
  • 一个类代表一个对象的集合，这些对象共享
2. 类的接口
  • 提供类的外部视图，通过使用接口使用类的功能
3. 类的实现
  • 类的内部视图，实现接口声明等
4. 类的关系
  • 聚合（整体-部分）关系：一个类里面声明了另一个类的对象作为类的全局属性
  • 关联关系：一般是一种语义依赖，一个类的方法里与另一个类产生了关系
    • 重数Multiplicity
  • “is a”关系：就是继承
5. 继承

  • 可以用继承表达特化（Specialization）和泛化（generalization）

  • 具体类和抽象类

  • 多态

对象

1. 定义相关
  • 指一个实体，必须有以下几个要素：状态+行为+标识
2. 对象的状态
  • 一般泛指对象之中的数据（变量）
  • 系统的所有状态都被对象封装
3. 对象的行为
  • 对象之间的作用和反应，为了引起一些反应（可能导致状态的变化）
  • 对象不同状态下会有不同的行为
4. 主动对象和被动对象
5. 对象的标识
  • 用于区分不同的对象，一般就是用不同的变量名标识
6. 对象之间的关系

  • 连接

  • 聚合
