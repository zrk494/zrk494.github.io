传输层

1. 传输层服务

1.1 概述

传输层协议为运行在不同主机之上的进程提供了一种逻辑通信机制（端到端）

1.2 逻辑通信机制

可理解为不同主机直接相连通信，在通信中无需考虑承载报文的物理设施细节

1.3 与网络层的关系

网络层提供不同主机间的逻辑通信

传输层提供不同主机的不同应用进程间的逻辑通信

传输层依赖于网络层提供的服务

2. 多路复用+多路分解

2.1 多路复用

2.1.1 是什么

应用层所有的应用进程都可以通过传输层再传输到网络层

2.2 多路分解

2.2.1 是什么

传输层从网络层收到数据后交付指定的应用进程

2.2.2 如何工作

主机收到IP数据报（包括源IP，目的IP，源端口，目的端口）—–>主机收到数据段后，传输层提取IP和端口号，再将数据段导向套接字

2.2.3 无连接的分用UDP

二元组标识套接字：（IP，端口号）

2.2.4 面向连接的分用TCP

四元组标识套接字

3. UDP（无连接的用户数据报协议）

3.1 特征

  • 不需要建立连接，收到报文之后也不需要确认（延时很小
  • 基于IP协议，增加了复用/分用，以及简单的错误校验这些内容，实现简单
  • UDP使用最大努力交付，不保证交付数据可靠性，UDP段可能丢失或者非按序到达
  • 无拥塞控制，更好地控制发送时间和速率
  • 面向报文
  • 首部开销小（8字节）
  • 通过应用层的可靠性机制可能实现UDP上的可靠数据传输

3.2 报文格式

首部字段（源端口+目的端口+UDP长度+UDP检验和）+数据字段

3.3 UDP校验和（若有错则直接丢弃报文）

发送方：将该段的内容视为16位整数，计算所有整数的和，得到的和按位取反，就是校验和，放在校验和字段之中

接收方：计算收到的字段的校验和，与校验和字段对比

进程

1. 基本概念

1.1 任务（task）和作业（job）的区别

简单来说，作业就是任务的一个实例（类比类和对象的概念）

1.2 程序和进程的区别

程序是被动的，进程是活动的，当程序被载入内存的时候，这个程序就成了进程

1.3进程的结构形式***

1.3.1代码块（文本段）

包括程序代码

1.3.2栈

包括函数参数，返回值，局部变量

1.3.3数据段

包含全局变量

1.3.4堆

程序运行过程之中动态分配的内存

1.4 进程控制块（PCB process control block ==TCB task control block）

进程创建的时候，系统新建PCB来保存进程的当前状态信息，以便于操作系统管理和控制

包括8个内容: 进程ID，进程状态，进程计数器，CPU计数器，CPU调度信息，内存管理信息，记账信息，I/O状态信息

1.5 进程状态

5种状态: new, ready, running, waiting, terminated

2. 进程的调度

2.1 进程调度队列

CPU往往要在多个可用进程之间快速切换，此时调度程序要选择其中一个进程运行，这些进程就存放在队列之中

队列包括设备队列（要等待特定I/O的队列）和就绪队列（暂存等待运行的进程）

2.2 程序调度

2.3 调度程序

分成3种：长期调度程序，中期调度程序，短期调度程序

2.4 上下文切换

CPU在切换到另一进程时，要保存当前进程的状态，并恢复另一进程的状态

3. 进程操作过程

3.1 进程创建

  • 进程是树状结构，所以有子进程和父进程
  • PID是进程唯一的标识符（操作系统通过PID来识别进程）
  • 一个进程可以创建另一个进程（即创建一个子进程）
  • 进程创建选项
  • 创建过程 在系统内部创建程序块—->分配内存—->载入可执行文件—->初始化程序
  • （以UNIX系统为例）fork()函数

3.2 进程终止

  • 进程完成之后用exit()删除自身并终止运行
  • 可以用wait()
