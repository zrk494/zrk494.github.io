发表于 2021-04-27 更新于 2021-05-24

网络层（两大方面，数据平面和控制平面 ）

1. 概述

1.1 数据平面

包括网络层中路由器的功能，他的功能决定到达路由器输入链路之一的数据报如何转发到路由器的一条输出链路

1.2 控制平面

包括网络范围的逻辑，控制数据报沿着从源主机到目的主机的端到端路径中路由器之间的路由方式。

补充

  • 为了保证转发效率，数据传输的信道是不可靠的
  • 每个主机和路由器都运行带着网络层的协议
  • 路由器会检验每个经过它的IP数据报，但是只检测头部
  • 网络层为发送端到接收端的数据报传送通道提供了相关服务

2. 提供的服务

2.1两个核心功能

2.1.1 转发（forwarding）

  • 将分组从路由器的输入端口运输到合适的出口

  • 转发表（包括地址和输出链路）辅助完成上述的过程

2.1.2 路由（routing）

  • 通过路由算法计算从H1（起点）到H2（终点）的路径集合

  • 用路由表记录路由信息（如何转发分组）

2.2 其他的服务

2.2.1 连接建立（不是所有的都是必要的）

与传输层的区别:

  • 对于网络层：所有线路上的设备都参与连接，是主机之间的连接

  • 对于传输层：是两个应用进程之间的连接

2.2.2 网络服务模型 （为传输层提供的服务）

  • 不同类型的网络提供的服务是不同的（例如：因特网停供的是一种尽力服务），主要分成以下两种

  • 无连接服务

    • 每个分组时独立确定传输路径的，不同分组经过的路径不同

  • 连接服务

    • 建立逻辑连接（确认了一条传输路径）
    • 后续分组都沿着此路径传输
    • 结束后要拆除连接

2.4 数据报（报文）网络（无连接服务的代表）

4. ip协议（因特网的网络层）

4.1 概述

  • IP协议用于处理和转发数据分组（根据路由表）
  • 包括如何寻址，如何处理分组等

4.2 数局报格式（两部分：首部（对应下图option及之前的内容）+数据）—–以IPv4为例

  • 版本号 4位
  • 首部长度 4位，以4字节为单位
  • 服务类型 8位，一般情况下不使用
  • 数据报总长度 16位，以字节为单位
  • 标识，标识位，片偏移（详见IP分段）
  • 寿命（生存时间TTL） 8位，表示IP分组在网络中可以通过的路由器数，路由器转发一次TTL减1，while（TTL=0），该分组被丢弃
  • 协议 8位，表示IP分组封装的是哪一个协议的数据包
  • 首部检验和 16位，同UDP的差错检测算法，在每次转发过程中都要重新计算，重新校验
  • 源IP和目的IP地址 各32位
  • 选项字段（很少用）+填充（补齐）
  • 数据

4.3 数据段分片（IP分段）

4.3.1 最大传输单元MTU

  • 一个链路层贺能承载的最大数据量

  • 因为链路层的协议不同，MTU的大小不同

  • 一个较大的IP分组向MTU小的链路转发时，可以（不是一定）将IP分段，以封装进入链路层帧（定义详见数据链路层笔记）之中（为什么要分片）

4.3.2 数据分片之后的事项

  • 分片到达目的主机之后再进行重组

  • 为了让主机知道如何将这些分段重组，IP数据报之中的一些字段包含了标识以及确定顺序的信息

    • 标识字段（ID）

      • 16位，标识IP分组（IP协议中存在一个计数器，每产生一个IP分组，计数器+1，作为IP分组的标识）
      • 和其他一些分组的信息一起才可以唯一标识一个IP分组

    • 标志位

    • 3位，包括DF（Don’t Fragment，表示是否可以分片）和MF字段（表示是否是分片的最后一片）

    • 片偏移

      • 13位，指一个IP分片封装的IP数据的相对偏移量
      • 以8字节为单位
4.3.3 IP分片的过程及相关计算（设原IP分组总长位L，MTU为M）
  • IF（L>M）则需要分片
  • 分片后每个分片会复制原IP的标识

4.4 IP编址

4.4.1概述
  • 接口
    • 主机与物理链路之间的边界
  • IP地址
    • IP地址长度为32比特
    • 用点分十进制记法书写
  • 引入IP子网的概念
    • 将IP地址分成两部分：网络号（高位）+主机号（低位）
    • IP子网内的设备网络号相同
    • 路由器一般为一个IP子网的边界
4.4.2 IP编址方式，地址划分

  • 有类IP地址

    • A类， 前8位为网络号
    • B类， 前16位为网络号
    • C类， 前32位为网络号
    • D类
    • E类

  • IP子网划分

    • 有类IP地址划分不平衡，像是ABC类网络太大，会存在浪费现象，用子网划分的方式进一步对大的网络进行划分

    • IP地址分成3部分：网络号+子网号+主机号（将之前的高位主机号的一部分划分成SubID）

    • 子网掩码

      • 用于确定是否划分了子网，以及使用多少位去划分子网
      • 格式同IP，32位，点分十进制，NetID和SubID全0，HostID全1

    • 子网地址=IP分组的目的IP地址与子网掩码按位与（子网地址标识一个子网，就是子网开始的IP地址，让路由器明白将IP分组转发到哪一个子网）

    • 子网地址+子网掩码可以准确确认一个子网的大小，并且可以用这样的形式表示一个子网

  • CIDR（新的地址模式，称为无类域间路由）

    • 消除了传统的ABC三类地址的界限，将NetID和SubID合并为Prefix（前缀），Prefix可以为任意长度

    • 与子网划分的模式相结合，便于划分IP地址

    • 无类地址的格式a.b.c.d/x，x为前缀长度（位长），可直接表示一个子网

      • 例 子网 201.2.3.64，255.255.255.192 的无类地址 201.2.3.64/26

    • 此种模式的优点

      • 提高地址分配效率
      • 提高路由效率
        • 此种方式可以将多个子网聚合成为更大的子网，形成层级化的编址
        • 路由聚合，路由表记录更大的子网的无类地址（记录更高层的地址），减少不必要的信息的记录
        • 连续的几段地址才可以聚合成更大的子网
        • 补充 最长前缀匹配优先
4.4.3 主机如何获取IP地址（DHCP协议）

  • ”硬编码“，静态配置（后续补充完吧，大概）

    • 默认网关 某一个路由器与一个子网相连的一个接口的IP，作为这个子网之中所有主机的默认网关

  • 动态配置（动态主机配置协议DHCP）

    • 动态的从DHCP服务器获取IP地址+子网掩码+默认网关地址+DNS服务器名称及其IP地址

    • 地址是可以重用的（一个主机下线之后他的地址可以被分配给其他的主机）

    • 配置过程

      1. 找到DHCP服务器（主机通过广播DHCP发现报文，此时主机的IP地址为0.0.0.0，此条广播只有DHCP服务器会响应）
      2. DHCP服务器响应（DHCP服务器通过广播发送一个DHCP提供报文，报文包括服务器的IP地址+端口号+拟分配IP地址+IP地址有效时间）
      3. 请求IP地址（主机通过广播发送DHCP请求报文）
      4. DHCP确认（DHCP服务器向主机发送DHCP ACK，表示可以给主机提供IP地址）
      • 补充，第3步还是通过广播发送的原因是主机告诉其他DHCP服务器已经选择好了用哪个DHCP服务器了，其他DHCP服务器会收回拟分配的IP地址

    • DHCP协议是在应用层实现的（运输层使用UDP协议）
