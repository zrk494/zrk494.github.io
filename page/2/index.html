网络层（两大方面，数据平面和控制平面 ）

1. 概述

1.1 数据平面

包括网络层中路由器的功能，他的功能决定到达路由器输入链路之一的数据报如何转发到路由器的一条输出链路

1.2 控制平面

包括网络范围的逻辑，控制数据报沿着从源主机到目的主机的端到端路径中路由器之间的路由方式。

补充

  • 为了保证转发效率，数据传输的信道是不可靠的
  • 每个主机和路由器都运行带着网络层的协议
  • 路由器会检验每个经过它的IP数据报，但是只检测头部
  • 网络层为发送端到接收端的数据报传送通道提供了相关服务

2. 提供的服务

2.1两个核心功能

2.1.1 转发（forwarding）

  • 将分组从路由器的输入端口运输到合适的出口

  • 转发表（包括地址和输出链路）辅助完成上述的过程

2.1.2 路由（routing）

  • 通过路由算法计算从H1（起点）到H2（终点）的路径集合

  • 用路由表记录路由信息（如何转发分组）

2.2 其他的服务

2.2.1 连接建立（不是所有的都是必要的）

与传输层的区别:

  • 对于网络层：所有线路上的设备都参与连接，是主机之间的连接

  • 对于传输层：是两个应用进程之间的连接

2.2.2 网络服务模型 （为传输层提供的服务）

  • 不同类型的网络提供的服务是不同的（例如：因特网停供的是一种尽力服务），主要分成以下两种

  • 无连接服务

    • 每个分组时独立确定传输路径的，不同分组经过的路径不同

  • 连接服务

    • 建立逻辑连接（确认了一条传输路径）
    • 后续分组都沿着此路径传输
    • 结束后要拆除连接

4. ip协议（因特网的网络层）

4.1 概述

  • IP协议用于处理和转发数据分组（根据路由表）
  • 包括如何寻址，如何处理分组等

4.2 数局报格式（两部分：首部（对应下图option及之前的内容）+数据）—–以IPv4为例

  • 版本号 4位
  • 首部长度 4位，以4字节为单位
  • 服务类型 8位，一般情况下不使用
  • 数据报总长度 16位，以字节为单位
  • 标识，标识位，片偏移（详见IP分段）
  • 寿命（生存时间TTL） 8位，表示IP分组在网络中可以通过的路由器数，路由器转发一次TTL减1，while（TTL=0），该分组被丢弃
  • 协议 8位，表示IP分组封装的是哪一个协议的数据包
  • 首部检验和 16位，同UDP的差错检测算法，在每次转发过程中都要重新计算，重新校验
  • 源IP和目的IP地址 各32位
  • 选项字段（很少用）+填充（补齐）
  • 数据

4.3 数据段分片（IP分段）

4.3.1 最大传输单元MTU

  • 一个链路层贺能承载的最大数据量

  • 因为链路层的协议不同，MTU的大小不同

  • 一个较大的IP分组向MTU小的链路转发时，可以（不是一定）将IP分段，以封装进入链路层帧（定义详见数据链路层笔记）之中（为什么要分片）

4.3.2 数据分片之后的事项

  • 分片到达目的主机之后再进行重组

  • 为了让主机知道如何将这些分段重组，IP数据报之中的一些字段包含了标识以及确定顺序的信息

    • 标识字段（ID）

      • 16位，标识IP分组（IP协议中存在一个计数器，每产生一个IP分组，计数器+1，作为IP分组的标识）
      • 和其他一些分组的信息一起才可以唯一标识一个IP分组
